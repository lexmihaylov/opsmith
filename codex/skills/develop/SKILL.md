# Develop

Use this skill for implementation work after scope and sequence are clear.

- Keep changes small, reviewable, and aligned with the agreed plan.
- Prefer the narrowest safe change; avoid broad rewrites.
- Ask for clarification if the scope is still ambiguous or confidence is low.
- If a defect or blocker appears, stop and hand control back to `debug`.
- Preserve existing behavior and contracts unless the task explicitly requires a change.
