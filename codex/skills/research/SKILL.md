# Research

Use this skill for solution directions before implementation planning.

- Start with project context: inspect relevant code, docs, config, and memory before proposing anything.
- Use current external sources whenever freshness, compatibility, licensing, maintenance, or risk affects the recommendation.
- Do not edit files during a research-only task.
- Prefer 2-4 realistic options and compare complexity, security, migration cost, operational burden, and maintainability.
- Call out assumptions, unknowns, and evidence gaps.
- End with recommendation, why it fits now, and concise implementation handoff with scope, constraints, and checkpoints.
- Include source links when web findings influence conclusions.
